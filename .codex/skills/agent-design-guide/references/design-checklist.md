# Agent 设计清单

## 1. 需求定标
- 用户目标是什么？
- 成功标准是什么？
- 允许的错误类型是什么？
- 时延、成本、准确率、可解释性哪个优先？
- 是否涉及金融、医疗、法律、隐私、生产写操作？

## 2. 核心工作流模板

```text
输入
→ 路由/分类（可选）
→ 规划（可选）
→ 信息获取（RAG / 工具 / MCP）
→ 执行（单 Agent 或多 Agent）
→ 反思/校验（可选）
→ 输出
→ 监控、记录、学习（可选）
```

## 3. 模式选型问题
- 任务是否必须拆成多个顺序步骤？→ 提示词链
- 是否要根据输入切不同路径？→ 路由
- 是否存在可并发子任务？→ 并行化
- 是否需要先生成计划再执行？→ 规划
- 是否需要调用外部 API / DB / 发消息 / 写数据？→ 工具使用
- 是否需要标准化接入大量外部能力？→ MCP
- 是否需要从私有知识中检索？→ RAG
- 是否需要跨轮记住用户、任务或历史？→ 记忆管理
- 是否需要多个专业角色协作？→ 多 Agent / A2A
- 是否需要自动自检或二次打磨？→ 反思
- 是否要持续追踪目标偏差？→ 目标设定与监控
- 是否存在失败、超时、脏数据、工具异常？→ 异常处理与恢复
- 是否为高风险输出？→ Guardrails + HITL
- 是否要控成本控时延？→ 资源感知优化
- 是否有多个目标抢资源？→ 优先级排序
- 是否需要持续改进或发现新策略？→ 学习与适应 / 探索与发现

## 4. 默认架构建议

### 简单业务 Agent
- 单 Agent
- 路由（可选）
- 规划（简单任务可省）
- 工具使用
- 基础异常处理
- 基础监控

### 知识型 Agent
- 单 Agent 或 coordinator
- RAG
- 记忆管理
- 反思
- 评估与监控
- Guardrails

### 企业流程 Agent
- 规划
- 路由
- 工具使用 / MCP
- 目标监控
- 优先级排序
- 异常恢复
- 审计日志
- HITL

### 研究型/开放探索 Agent
- 多 Agent
- 推理技术
- 探索与发现
- RAG / 工具 / MCP
- 评估框架
- 资源感知优化
